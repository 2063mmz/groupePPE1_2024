Étant donné que nous avons quatre langues, il est nécessaire de définir les variables globales et les chemins de sortie, afin de configurer les termes de recherche et les chemins des fichiers HTML de sortie en fonction des différentes bases de la langue.

Étape suivante importante : définition de la page HTML et automatisation de la création et de l'organisation des informations, dans le but de générer l'en-tête du fichier HTML et la structure du tableau, afin d'afficher les résultats du traitement..

Lire le fichier d'URLs ligne par ligne, traiter chaque URL, extraire les informations nécessaires et les remplir dans le tableau HTML. Utiliser la commande curl pour obtenir le code d'état HTTP de l'URL, afin de déterminer si l'URL est accessible. Ensuite, extraire le codage des caractères de la page, en fonction du système d'exploitation utilisé.

Ici, c'est pour télécharger le contenu HTML de l'URL, gérer le codage et sauvegarder dans le répertoire spécifié.

Si la réponse HTTP ne contient pas d'information sur le codage, extraire celle-ci à partir des balises meta du HTML. Si le codage n'est pas en UTF-8, le convertir en UTF-8 (Pour éviter une conversion forcée en UTF-8, il est possible, dès le début lors de la sélection des URLs, de filtrer et d'exclure les sites ne fournissant pas d'information sur le codage).

Utiliser lynx pour convertir le contenu HTML en texte brut et effectuer le vidage du contenu textuel. def

Supprimer le contenu textuel inutile, afin de maintenir la propreté du texte. Définir les informations textuelles à supprimer. (En plus de certains symboles spéciaux, il faut également supprimer le contenu entre [] et ()).

Compter le nombre d'occurrences du mot-clé cible dans le texte nettoyé. occurrences=$(grep -o -i -E "$mot_pattern" <<< "$cleaned_contenu_dump" | wc -l) Pour grep -o -i -E "$mot_pattern" Recherche toutes les occurrences du motif défini dans mot_pattern dans le texte fourni, en ignorant la casse et en utilisant des expressions régulières étendues, et affiche uniquement les correspondances trouvées. Sur <<< est le 'Here String'. Cette partie fournit le contenu de la variable cleaned_contenu_dump comme entrée à la commande grep.

Extraire le contexte contenant le mot-clé cible.

Générer les informations de concordance pour le mot-clé, et les sauvegarder au format HTML.

Ajouter le résultat du traitement de l'URL actuelle comme une ligne dans le tableau HTML.

Si l'URL est inaccessible, l'enregistrer comme "Indisponible" et le marquer dans le tableau HTML.

Enfin, terminer! Écrire dans l'ordre pour fermer le tableau et les balises de la page HTML.
